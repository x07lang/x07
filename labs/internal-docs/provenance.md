# Provenance and signing

This document describes how release artifacts can be verified.

## Release manifests

Each release should include `dist/release-manifest.json`, generated by `scripts/build_release_manifest.py`.

The manifest records:

- The git SHA used to build the release
- The pinned lockfiles and their hashes
- The canonical schema files and their hashes

The release workflow also emits installer-facing manifests:

- `dist/release/x07-<version>-release.json` (`x07.component.release@0.1.0`)
- `dist/release/x07-<version>-bundle.json` (`x07.release.bundle@0.1.0`)
- `dist/channels.json` for bootstrap compatibility
- `dist/channels/<channel>.json` for channel-driven `x07up` installs

## Signing and provenance

Provenance inputs are recorded in `dist/release-manifest.json` (see `scripts/build_release_manifest.py`).
Checksums and attestation bundles are emitted into `dist/release/` before the component and bundle
manifests are finalized, so the release manifests can reference immutable hashes for the shipped assets.
