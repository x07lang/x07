Subject: one frontier agent (Claude Fable 5, via Claude Code), solving the
6-task `core` suite in Python and X07, judged by `runner.py` (bytes-out ==
expected for every vector). This is a PILOT: it calibrates the harness and
gives a first signal. It is not the comparative evidence the language bet
needs — see RUNBOOK.md for the scaled protocol and its predeclared decision
rule.

## Failure analysis (X07)

Both first-attempt failures (`top_word`, `rle_encode`) had a single root
cause: the agent wrote `==` for equality; x07 uses `=`. Every mainstream
language uses `==`, so model priors fight this convention on every fresh
generation. Notes:

- The error (`Unsupported: unsupported head: "=="`) came from the C-emit
  stage as a bare string — no did-you-mean, no pointer. The typecheck-stage
  unknown-callee path has suggestions now; the unsupported-head path should
  get the same treatment (follow-up).
- All other guessed conventions (`>`, `<`, `&&`, `||`, move/rebind
  discipline, slice-table APIs) were correct on first attempt — in part
  because the subject had warm in-session knowledge (see caveats).

## Tooling effect observations (new DX features, same session)

- `x07 doc` fuzzy lookup resolved a half-remembered name (`get_u8` →
  `std.bytes.get_u8`) in one query; before this feature the same query
  dead-ended with zero suggestions.
- The behavioral summaries surfaced the non-obvious
  `std.json.extract_path_canon_or_err` calling convention
  (`<json> 0x00 <path>` packed into one argument) without reading stdlib
  source. In the morning baseline session (before summaries existed), an
  equivalent unknown (`tokenize_words_lower` output format) cost a wrong
  program that ran silently and a dig through Rust reference sources.
- One solution (`upper_ascii`) was authored in x07text and converted with
  `x07 ast from-text`; it compiled and passed on first attempt.

## Caveats (read before quoting numbers)

1. n=1 model, n=1 session, 6 tasks. Nothing here is statistically meaningful.
2. Self-administered: the subject agent also built the tooling being
   evaluated, and had spent the whole session reading x07 docs — a cold-start
   agent would pay a substantially higher discovery cost than this pilot shows.
3. Substrate-class tasks only (pure bytes-in/bytes-out). No I/O, no
   multi-module projects, no dependency management — the places earlier case
   studies (x07dbguard) hit hard blockers.
4. Python numbers are effectively a ceiling for this comparison: there is no
   discovery cost in a language the model has saturated priors for.

## First-signal interpretation

Consistent with the strategic review: x07's correctness loop converges fast
(one fix round, driven by a deterministic error), but generation cost
(bytes ≈ 4x, discovery lookups before the first attempt) and prior-fighting
conventions (`=` vs `==`) are real taxes. The scaled run (RUNBOOK.md) decides
the language bet; this pilot only proves the harness and logs where the new
DX work moved friction.
